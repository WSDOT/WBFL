Introduction

The Fem2d engine is designed to be a low-level, high-performance component for 2D frame analysis. The interfaces for the model should be intuitive for anyone who has used a general-purpose finite element program like GT-STRUDL or NASTRAN. However, the number of features for Fem2d is much less than one of these comprehensive engines.

Analysis Features

The Fem2d engine supports a wide array of features for modeling 2D linear-elastic frames. Both axial and bending behaviors are modelled. Shear deformation is not modeled. The main features are as follows.

  • 2d linear-elastic prismatic beam elements (Members)

  • Model size is only limited to available memory

  • Member end releases for moment

  • Unlimited number of load cases (Loadings)

  • Supports many different types of loads including:

    • Concentrated loads applied to joints

    • Concentrated loads applied along members

    • Trapezoidally distributed loads applied to members

    • Axial and curvature strains applied to elements

    • Joint Displacements (support settlement) loads

  • Unlimited analysis points (POI's) along members (eliminates need to add joints where results are desired)

  • Simple event model for clients

  • File-based persistence of Model data

It is also important to note the features that you might want, but are not supported:

  • Automatic bandwidth reduction (you have to number your own nodes efficiently)

  • Master-Slave relationships

  • Plate and shell-type elements

  • Fine-grained event model (nice for GUI-based programs, but tough on performance)

  • Non-prismatic Members

  • Zero-size spring elements

  • Inclined supports
