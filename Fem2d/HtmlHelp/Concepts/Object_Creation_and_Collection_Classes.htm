Object Creation and Collection Classes

If you are an experienced component developer and have looked through the Fem2d component, you may have seen that there is only one object in the entire component that is "createable": the Model (a "creatable" object is an object that can be created in your program by calling CreateInstance in C++, or by using new in Visual Basic or scripting languages). This is by design. It does not make sense to create Members, Joints, or loads unless they belong to a Model.

A look at the class diagram for the Fem2d reveals that all objects (except Model) either already exist in the Model or are created by their collections. In fact, all of the many collections in the Fem2d have a factory method named Create that is used to create and insert a new thing in the collection.
