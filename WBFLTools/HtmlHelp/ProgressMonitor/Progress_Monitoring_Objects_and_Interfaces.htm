Progress Monitoring Objects and Interfaces

The progress monitor objects and interfaces in this section provide a generic way to add progress monitoring to your computational applications. The window shown above shows an instance of the ProgressMonitorWindow object found in this toolbox.

Computational engines that want to use progress monitoring must implement the ISupportProgressMonitor interface. Engines are responsible for sending status messages to the monitor, reporting percent complete, and checking to see if the computation was cancelled. This status information is sent back to a progress monitor by the engine via the IProgressMonitor interface.

The ProgressMonitorWindow object provides an implementation of IProgressMonitor that can be used to display a window, like shown above, and/or to dump progress messages to a file.
