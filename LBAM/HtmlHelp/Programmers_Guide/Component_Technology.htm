Component Technology

Software components are to software what integrated circuits (ICs) are to electronics: "black boxes" that encapsulate functionality and provide services based on a specification. By building software in this fashion, we can optimize reuse and reliability. Software components are designed to be reusable, even interchangeable. Also, software components can be tested individually and then be used to compose larger systems. This helps to increase overall system reliability and flexibility. For example, many of the analysis components in the LBAM system were built on top of the Fem2D component.

All of the WBFL components, including the LBAM system, make use of Microsoft's Component Object Model (COM, see http://www.microsoft.com/com/ ) as an architecture for creating software components. COM provides standard tools and guidelines for building and using software components on the Windows platform.

COM is a language-independent specification, hence one can use any programming language that supports COM to create and use COM objects. The components in the LBAM system were all written using the C++ programming language. The Microsoft Active Template Library (ATL) was used extensively to provide COM support for our C++ objects.

Refer to the Bibliography for a list of some of the books that we found useful for learning and using ATL and COM.
