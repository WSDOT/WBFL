Live Load Strategies

The LiveLoadModelResponse object is dependent, by default, on the BruteForceVehicularResponse object for computing the enveloped responses for each of the VehicularLoads in the model. This includes the job of moving the trucks along the superstructure in order to find the optimal solution. We understand that the logic used by the brute force object is simplistic and not very efficient, and that you are much smarter than we are, so we are giving you a chance to write a better one using your own strategy. Here are the steps:

  1. The first thing that you need to do is create your own super-duper live loader object that supports the same interfaces as BruteForceVehicularResponse.

  2. Next, intialize the object so that is uses same live load context as the LiveLoadModelResponse object.

  3. Then identify your new object to the LiveLoadModelResponse object by setting the Strategy property via its IEnvelopingStrategy interface.

  4. The live load system will now be ready to use with your new object.

Unfortunately, at present you cannot use the LBAMAnalysisEngine along with your own live load strategy. This is because the LBAMAnalysisEngine object performs all of the intializations in its Initialize function, and it does not allow alternate live load strategies to be introduced after it has performed its initialization. This means that you will need to perform all LBAM engine initializations manually. Refer the the source code for the Initialize method of the LBAMAnalysisEngine for the details of how to do this in C++.
