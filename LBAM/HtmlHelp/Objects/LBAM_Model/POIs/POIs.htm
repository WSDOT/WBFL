POIs

This collection interface is used to access the POIs in a Model. This collection behaves like a set.

POIs can be enumerated, or found by ID.

Note: All POIs in the collection must have unique ID's. This condition is checked when a POI is added to the collection, however, the container cannot enforce the condition if a POI is renamed and given a duplicate name while it is in the container. If a POI is renamed while it is in the container, and it changes the ordering of the container, it may corrupt the container and cause undefined results.

Properties

Count

Number of POIs in collection

Item

Returns a POI from the collection

_EnumElements

Returns a fast enumerator (hidden,restricted)

Methods

Add

Adds (appends) a POI to the end of the collection

Clear

Remove all POIs from collection

Find

Find a POI by ID

RemoveByIndex

Remove a POI using its relative position in the POIs collection

RemoveByID

Remove a POI using its ID

Events

OnPOIsChanged

Fired whenever a POI in the collection is changed

OnPOIsAdded

Fired whenever a POI is added to the collection

OnPOIsBeforeRemove

Fired just before a POI is removed from the collection

OnPOIsRenamed

Fired after a POI has been renamed

Interfaces

These interfaces are obtained via QueryInterface
