Stages

This collection interface is used to access the Stages in a Model. This collection behaves like a sequential list of geometric segments ordered from left to right.

  • There must be at least one Stage in the model.

  • Each Stage must have a unique Name.

  • The collection supports the standard IDispatch-based collection access interface as well as access to a type-safe, fast vtable interface through a fast enumerator.

  • Access to the list of Stages is zero-based.

  • Stages can be inserted, moved and removed from the sequence.

Properties

Count

Number of Stages in collection

Item

Returns a Stage from the collection

_EnumSegments

Returns a fast enumerator (hidden,restricted)

Methods

Add

Adds (appends) a Stage to the end of the collection

FindIndex

Returns the index of the Stage with a given name

Insert

Inserts a Stage to a specified location in the collection

MoveTo

Move a Stage from one location to another

RemoveByIndex

Remove a Stage using its relative position in the Stages collection

RemoveByName

Remove a Stage using its name

Reverse

Reverses the order of the Stages in a collection

Events

OnStagesChanged

Fired whenever a Stage in the collection is changed

OnStagesAdded

Fired whenever a Stage is added to the collection

OnStagesBeforeRemove

Fired just before a Stage is removed from the collection

OnStagesMoveTo

Fired whenever MoveTo is called

OnStagesCopyTo

Fired whenever CopyTo is called

OnStagesReverse

Fired whenever Reverse is called

Interfaces

These interfaces are obtained via QueryInterface
