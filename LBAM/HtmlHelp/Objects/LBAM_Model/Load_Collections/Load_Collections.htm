Load Collections

All loads in an LBAM (PointLoads, DistributedLoads) are stored in collections that behave in an identical manner. The name of the collection is the same as the load, except with a 's' appended (e.g., "PointLoads", "TemperatureLoads", etc..). For sake of brevity, this section provides documentation for all load collections in the LBAM.

Loads are stored and retrieved using a unique key (aka ID) that is generated by the collection when the Load is inserted. Loads can also be sorted and filtered using a Stage name, and a LoadGroup name. When Loads are retrieved from the collection, they are returned in a special wrapper class called a LoadItem (e.g., "PointLoadItem", "TemperatureLoadItem", etc.) This wrapper is used to hold the load's context information in the container (ID, Stage, LoadGroup) along with the load itself. This approach decouples container information from the actual loads themselves. It also allows a single Load object to be used in more than one Stage/LoadGroup.

The collection supports the standard IDispatch-based collection access interface as well as access to a type-safe, fast vtable interface through a fast enumerator.

The Load ID's are issued by the container at creation time and remain read-only thereafter. This is done to insure referential integrity.

Properties

Count

Number of Loads in collection

Item

Returns a LoadItem from the collection using its index

Load

Set/Get and existing Load using its ID

_EnumElements

Returns a fast enumerator (hidden,restricted)

Methods

Add

Adds (appends) a Load to the end of the collection

Clear

Removes all Loads from the collection

Find

Find a LoadItem by ID

RemoveByIndex

Remove a Load using its relative position in the Loads collection

RemoveByID

Remove a Load using its ID

RemoveStage

Remove all Loads in the given Stage

Events

OnLoadsChanged

Fired whenever a Load in the collection is changed

OnLoadsAdded

Fired whenever a Load is added to the collection

OnLoadsBeforeRemove

Fired just before a Load is removed from the collection

Interfaces

These interfaces are obtained via QueryInterface

IStructuredStorage2

Persistence of LBAM data
