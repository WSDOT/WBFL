Getting Started With The

WBFL Longitudinal Bridge Analysis Model (LBAM)

Welcome to the on-line documentation for the WSDOT LBAM system. The WBFL LBAM system is a group of Windows-based COM1 components used for describing and performing a structural analysis of bridge superstructures.

If you are reading this for the first time, it is likely that you have a lot of questions right now about what the LBAM system is, and how you might use it. The first thing that you should understand is that this is a fairly large software system - It took over two man-years to design and program it, so don't be surprised or frustrated if you don't fully understand the entire system after looking at it for a couple of hours. The good news is that the system is designed to be easy to understand for bridge engineers and will save you many man-months in your implementation.

In our opinion, the best way to get acquainted with the system is as follows:

  1. Some prerequisites: Get familiar with COM or Visual Basic. See the What Do I Need To Know section for more details.

  2. Read the Getting Started and Basic Concepts sections of this document.

  3. Print out all of the Object Models and keep them nearby. They are the road map to the system.

  4. Browse the API Documentation. This describes the details of the system and is the best way to get closely familiar with the capabilities of the system.

  5. Examine the Sample Problems. The VBA2 examples written in Excel VBA are probably the best way to learn the system. The context-sensitive help and Autocomplete features in Excel are very helpful for learning the system's capabilities.
