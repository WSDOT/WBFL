Analysis Results Objects

Nearly all of the interfaces that compute and return structural responses for the LBAM return them in collections of anaysis results objects. Typically, the interfaces take an array of POI ID's as input and return a collection of results objects in the same order.

The reason that the response interfaces allow you to request collections of results is to increase performance. This gives the engine a chance to optimize its computations. You will always get a faster response from the analysis engines if you request a collection of results - rather than asking for one at a time.

There are many different types of results objects returned by the analysis engines. They vary depending on the reponse being requested. Refer to the API documentation for detailed descriptions of the anlaysis results objects.
