Using the LBAMAnalysisEngine

Once an LBAM model is built, it can be analyzed using one of the many analysis engines available with the LBAM system. The creation of analysis engines and their associated context objects can be an arduous task, especially for the beginning COM programmer. Fortunately the LBAMAnalysisEngine object provides a simplified way to create a default suite of analysis engines that will meet most needs. Refer to the Programmers Manual if you want to learn how to initialize individual analysis engines.

Once initialized, the LBAMAnalysisEngine object provides easy access to interfaces for computing analysis results on the LBAM in question. An object model for the LBAMAnalysisEngine can be found here. Results can be computed for LoadGroups, LoadCases, influence lines, live loads, and LoadCombinations as well as other quantities. Refer the to API documentation for the LBAMAnalysisEngine for a detailed list of the interfaces it supports and links to a detailed definition for each interface.

The simple example in the Getting Started chapter provides an example of using the LBAMAnalysisEngine using VBA. Refer to the other example problems and the testing code for the LBAM for examples of using C++ and other languages.
