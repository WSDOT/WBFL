Requesting Model Results - POI's

You must first place a POI at a location before you can request results at it. POI (point of interest) objects are used to define locations on the model where you can later request results from analysis engines. POI's can be placed at any location on the LBAM model. Each POI is given a unique ID number as a key. This key, or more often an array of keys, is then passed into the appropriate analysis engine when results are requested.

Note that NO POI's are placed on the model by default - you must place them manually. Or, you can use the GeneratePoisAlongSuperstructure method of the LBAMPOIUtility object to place POI's at evenly-spaced increments along the superstructure. This method is very handy for creating POI's at tenth points, as recommended by LRFD.
