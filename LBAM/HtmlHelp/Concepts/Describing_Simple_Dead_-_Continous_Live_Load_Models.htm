Describing Simple Dead - Continous Live Load (SDCL) Models

The LBAM is capable of modeling different types of stage-dependent behavior. A common scenario in precast concrete and steel bridge design is to model the bridge using simple (pinned) supports for dead load and continuous supports for live load. A typical scenario is shown in the following figure.

The analysis takes place over three stages.

Stage 1 - Bare Girders are placed on the piers. Dead load of girders are reacted by pinned supports.

Stage 2 - Slab and diaphragm is poured. Added dead weight of wet slab is applied to simply-supported girders.

Stage 3 - Diaphram is cured causing the girder to act continuously. Slab is cured and girder behaves as a composite section. Live load analysis is normally applied in this stage.

Modeling

The LBAM Component was designed to handle modeling this type of behavior. A basic outline to create such as model is as follows:

1.

Define three stages (e.g., "Stage 1", "Stage 2", "Stage 3").

2.

Lay out Spans and Supports.

3.

Create two SuperstructureMembers with lengths equal to the span lengths and add them to the model.

4.

The pinned connection for a fixed-length Support can be modeled in either of two ways:

a) If the top of the column is not pinned (integral bent): Release the right end of the first SuperstructureMember and the left end of the second SuperstructureMember. Set the release removal stage for each to "Stage 3".

b) If the top of the column is to be pinned: Use the TopRelease property of the Support to create a hinge. Then, release the right end of the first SuperstructureMember only. Set the release removal stage to "Stage 3".

5.

Define the section properties for the SuperstructureMembers for each stage of the analysis.

6.

Define the incremental self weight loads for each stage of the analysis.

7.

Compute cumulative force, stress, and deflection results as desired.

 

Tip 1: If you are modelling a zero-length support using bcPinned or bcRoller boundary conditions, you cannot release both SuperstructureMembers framing into the Support. If you do the model will be unstable.

Tip 2: It probably makes no sense to use bcFixed boundary conditions for zero-length supports. This effectively fixes rotation at the support point which would isolate any interaction between adjacent spans.

This was a brief overview of how SDCL structures can be modeled using the LBAM. You can also define temporary supports to model falsework used during construction. Refer to the Samples for more examples of models using staged construction.
