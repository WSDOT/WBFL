Analysis Context Objects

Analysis context objects are used to decouple the live load and load combination analysis engines in the LBAM system from the LBAMModel, and from each other, as much as possible. This is done by making an analysis engine dependent a separate object, a context object, that implements specific interfaces. For example, refer to the object model for the LBAM Live Load Engine. Note that the live load analysis objects are not directly dependent on the LBAMModel or the LBAMAnalysis packages, but only on the interfaces that are supported by the VehicularAnalysisContext object.

There are two analysis context objects in the LBAM system, the VehicularAnalysisContext , and the LoadCombinationAnalysisContext. for the LiveLoader and LoadCombiner packages, respectively.

Analysis context objects make it possible to reuse the objects in the LBAMLiveLoader and LBAMLoadCombiner packages to analyze another system. All that one needs to do is to create a new context object that implements the interfaces given in the specifications for the desired context. For an example of how to do this in C++, refer the the TestUsingBogusContext and BogusCombinationContext C++ objects in the LBAMLoadCombiner\LoadCombinerTest project. These objects are used to test the load combination engine separated from other objects in the LBAM system.
