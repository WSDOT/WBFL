Introduction

The WBFL LBAM system is a group of Windows-based COM components that are used to describe and analyze bridge superstructures. LBAM's are an idealized, two-dimensional representation of a bridge using beam members, distribution factors, and idealized loads.

The engine is designed to be high-performance while allowing a wide variety of clients to work with it. This is done by providing Interfaces to any client that can call a COM server via the IUnknown or IDispatch interfaces. The LBAM component itself is a COM-based component that is designed to be used both by vtable-based clients like C++ and Visual Basic as well as scripting (IDispatch-based) clients like VBA, JScript, and VBScript. This means that the engine will work for a wide array of programing environments such as:

  • Visual C++

  • Visual Basic

  • Visual Fortran

  • Applications that support VBA, such as: Excel, Visio, AutoCAD, MS-WORD

  • JavaScript, VBScript, MathCad

  • and many more!

Features

The LBAM supports a wide array of features for modeling 2D bridge superstructures and supports. Note that piers are modelled as a single, in-plane, column. Both axial and bending behaviors are modelled. Shear deformation is not modeled. The main features are as follows.

  • Modeling of multispan, continuous 2D longitudinal bridge structures

  • Model size is only limited to available memory

  • Modeling of simplified (zero-length) supports or fixed-length columns

  • Superstructure and support members are modeled by an unlimited number of prismatic beam segments

  • Stage-Dependent Analysis allowing time-dependent changes in

    • Section stiffness properties and stress coefficients

    • Loads (permanent or temporary)

    • Removal of in-span hinges

    • Removal of Temporary Supports

  • Unlimited number of load cases.

  • Generic loading heirarchy including Load Groups, Load Cases and Load Combinations

  • Many different types of basic applied loads including:

    • Concentrated loads

    • Trapezoidally distributed loads

    • Temperature Loads (lniform or gradient)

    • Support Settlement Loads

    • Generalized Member Strains (member misfit)

  • Generic description of live load models. Allows description of LRFD HL-93, LFD design live loads, and user-defined permit loads.

  • Generic description of load combinations. Allows description of all LRFD and LFD load combinations.

  • Separate force and deflection analysis models to support LRFD requirements.

  • Unlimited analysis points (POI's) along members

  • Persistance of model data to XML

  • And much more!
