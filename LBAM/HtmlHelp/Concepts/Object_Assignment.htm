Object Assignment

Objects in the LBAM are always assigned by reference. This can be very convenient, but also confusing.

For instance, if you have a four span structure with four identical spans, you can model this by simply creating one Span object and add it to the Spans collection four times. But then what if you want to change the length of Span 1 only? If, in our example, you get Span 1 from the collection and change its length, you will find that all four spans have been changed - not what you wanted. In order to perform what you need, you need to create a new Span object (either by cloning the old Span 1, or using a factory), and then Insert it into the model.
