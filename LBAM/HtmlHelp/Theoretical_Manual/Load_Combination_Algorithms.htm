Load Combination Algorithms

Load combinations are computed in the LBAM by factoring and summing responses from LoadCases and LiveLoadModels. Load combinations consist of an envelope of maximum values and minimum values for a given force effect.

Application of LoadCombination Factors

The live load portion of the load combination is formed by simply factoring the appropriate live load envelope values.

Application of LoadModifier Factors

LoadModifier factors allow load combinations to be modified for individual components in the structure. Each Span, Support, and TemporarySupport in the LBAMModel has its own set of max and min LoadModifiers which vary depending on LoadCombinationType. This gives the LBAM system the capability to model the h factors in LRFD (See LRFD 1.3.2 and 3.4.1).

Max and min LoadModifiers are applied to the LoadCase portions of the combinations using the algorithm shown in the Table in the previous section.

For live loads, the max factor portion of the LoadModifier is always applied to the combination. This is due to the fact that the enveloped live load value will always serve to maximize the effect in question.

Application of Live Loads

One or more LiveLoadModelTypes can be incorporated into a load combination. Whereas live load responses are the resulting envelopes of live load models, the results from each live load model type associated with a load combination are summed to produce the overall live load response.

Using an example to clarify, suppose you want to model a bridge loaded with the AASHTO LRFD HL-93 Live Load Model and Pedestrian live loading. The HL-93 Live Load is the resulting envelope of several truck and lane load combinations. We want to add the pedestrian load result to this envelope. To accomplish this, put the HL-93 live load in the lltDesign live load model and pedestrian live load in the lltPedestrain live load model. Then include both the lltDesign and lltPedestrain live load models in the load combination.
